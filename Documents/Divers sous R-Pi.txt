Le 30-10-2013 Le mot de passe de la R-Pi est changé


****** Connection Wifi *********
Renseigner les champs de connexion et attribuer une priorité (incrément de 10) pour avoir une connexion automatique en fonction du réseau détecté



*** FTP ***********
Utiliser FileZilla pour tranférer les sources sur la R-Pi.
(Connection SFTP: @IP port 22 Identifiant et mdp de la session active)

Depuis linux (Ubuntu):
sftp pi@adresse_IP puis saisir le mot de passe



*** SSH *****************
Depuis linux (Ubuntu)
ssh pi@adresseIP puis saisir le mot de passe



**** Compiler ***********
Avec VNC, se rendre dans le dossier source.
Compiler les sources avec:
 gcc -o nom_prog -Wall fichiers.c fichiers.h

Lancer le programme:
 ./nom_prog



****** Pour débugger **********
Compiler:
gcc -g test.c -o test

Lancer le programme dans le debugger:
gdb ./test

Vous allez apercevoir une invite de commande commençant par:
(gdb)

Tapez la commande :
run

Admettons maintenant que vous vouliez savoir quelle fonction a appelé quelle autre fonction depuis le début du programme jusqu'à l'arrivée de l'erreur.

Tapez la commande:
bt
